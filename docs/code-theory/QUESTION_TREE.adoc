== Phase 0 — Bounded Context

* *System*: Mistral Vibe — a Python 3.12+ CLI coding assistant published as
  the `mistral-vibe` package and the `vibe` / `vibe-acp` entry points
  (`pyproject.toml:1`, `pyproject.toml:3`, `pyproject.toml:6`).
* *Repository layout*: `vibe/core` (engine), `vibe/cli` (Textual TUI),
  `vibe/acp` (Agent Client Protocol server), `vibe/setup` (first-run
  wizards), `tests/` (`AGENTS.md:5`).
* *Scope of this recovery*: the entire product. ACP server and CLI are
  treated as two surfaces of the same bounded context because they
  share the engine, the configuration model, the tool registry, and the
  session log (`vibe/acp/acp_agent_loop.py`, `vibe/cli/cli.py`).
* *Not in scope*: the hosted Mistral AI Platform, the Vibe Code web
  product on the other side of `/teleport`, and any IDE clients that
  speak ACP (Zed, JetBrains, Neovim).

== Q3 — What is the architecture of this bounded context?

=== Q3.1 — Introduction and goals

`[ANSWERED]` Mistral's open-source CLI coding assistant; provides a

=== Q3.2 — Architecture constraints

`[ANSWERED]`

* Language and runtime: Python ≥ 3.12 (`pyproject.toml:6`).
* Package manager: `uv` is mandatory for development

* Strict typing: Pyright in strict mode gates CI (`AGENTS.md:46`); no

* Lint/format: Ruff is mandatory (`AGENTS.md:14`); no relative imports
  (`AGENTS.md:47`).
* TUI runtime: Textual 8.2.4 is pinned (`pyproject.toml:48`, ≈ near
  `textual==8.2.4`).
* External wire protocols: ACP 0.9.0 is pinned
  (`pyproject.toml`, dependency line `agent-client-protocol==0.9.0`),
  MCP ≥ 1.14.0 (same file), Mistral SDK 2.4.4 (same file).

=== Q3.3 — Context and scope

`[ANSWERED]` External actors and systems are enumerated in Q2.1.
Outbound traffic targets:

* Mistral chat-completions API (`vibe/core/llm/`).
* Mistral Transcribe and TTS APIs (`vibe/core/transcribe/`,
  `vibe/core/tts/`).
* Mistral datalake events endpoint (`vibe/core/telemetry/`).

Inbound: stdin/stdout (terminal), or stdio framed JSON-RPC (ACP).

=== Q3.4 — Solution strategy

`[ANSWERED]`

* *Hexagonal-style ports*: abstract interfaces use a `_port.py` suffix
  (`AGENTS.md:24`).
* *Pydantic-first data*: configuration and external data are validated
  through Pydantic models (`AGENTS.md:51-58`,
  `vibe/core/config/_settings.py`).
* *asyncio orchestration*: the agent loop and tool execution run on
  asyncio; HTTP via `httpx.AsyncClient` (`AGENTS.md:64-69`).
* *Streaming-by-default*: tool and model events flow as
  `AsyncGenerator[Event, None]` (`AGENTS.md:67`).

=== Q3.6 — Runtime view

`[ANSWERED]` Conversation turn lifecycle:

. CLI/ACP receives input, normalizes it, and dispatches to the
  `AgentLoop` (`vibe/core/agent_loop.py:221`).
. AgentLoop renders the system prompt
  (`vibe/core/config/_settings.py:725-734`), composes available tools
  through the tool manager (`vibe/core/tools/manager.py`), and calls
  the active LLM backend (`vibe/core/llm/backend/`).
. The model response is streamed; tool calls are extracted and either
  auto-allowed or routed through the permission system
  (`vibe/core/tools/permissions.py`, `vibe/core/tools/base.py`).
. Tool execution yields events streamed back to the UI.
. After a final assistant message, post-turn hooks run
  (`vibe/core/hooks/`).
. Messages and metadata are appended to the session log
  (`vibe/core/session/session_logger.py:34`).

Auto-compaction triggers when the cumulative input tokens cross
`active_model.auto_compact_threshold`
(`vibe/core/config/_settings.py:374`).

=== Q3.7 — Deployment view

`[ANSWERED]`

* Local install via `uv tool install mistral-vibe`, `pip install`, or
  the install script (`README.md:36-58`).
* Single-process model: TUI + engine run in one Python process
  (`vibe/cli/entrypoint.py:166`).
* ACP server is a separate process spawned by the IDE
  (`vibe/acp/entrypoint.py`).
* GitHub Action runs `vibe -p` in a CI runner (`action.yml`).
* PyInstaller distribution is also produced (`pyinstaller/`,
  `vibe-acp.spec`).

=== Q3.8 — Crosscutting concepts

`[ANSWERED]`

* *Configuration*: `VibeConfig` (`vibe/core/config/_settings.py:498`)
  via `pydantic-settings`, merging defaults, `config.toml`, and
  `VIBE_*` env vars.
* *Trust*: `trusted_folders_manager`
  (`vibe/core/trusted_folders.py`) decides whether project-context
  files are loaded.
* *Permissions*: `ToolPermission` (ALWAYS / ASK / NEVER) declared per
  tool (`AGENTS.md` "Tools" section, `vibe/core/tools/base.py`); the
  active agent profile may override
  (`vibe/core/agents/models.py:49-95`).
* *Telemetry*: `vibe/core/telemetry/` posts events to
  `https://api.mistral.ai/v1/datalake/events` when
  `enable_telemetry=True` (`vibe/core/config/_settings.py:519`).
* *Tracing*: OpenTelemetry, opt-in via `enable_otel`
  (`vibe/core/config/_settings.py:531-533`).
* *Logging*: stdlib logging with `StructuredLogFormatter`, written to
  `~/.vibe/logs/vibe.log` (`AGENTS.md`, `vibe/core/logger.py`).
* *I/O safety*: `read_safe` family handles encoding fallbacks
  (`vibe/core/utils/io.py`, `AGENTS.md` "File I/O" section).

=== Q4.6 — Security

`[ANSWERED]`

* Trusted-folder gating prevents arbitrary project files from
  altering the assistant's behavior
  (`vibe/core/trusted_folders.py`,
  `vibe/cli/entrypoint.py:124-153`).
* Tool permission model: `ToolPermission` declared per tool, ASK by
  default for destructive ones (`vibe/core/tools/permissions.py`,
  `AGENTS.md` "Tools" section).
* Sensitive-file blocklist applies to `read_file` and `grep`
  (mentioned in `AGENTS.md`, enforced in `vibe/core/tools/builtins/`).
* Browser sign-in flow uses PKCE-style exchange with payload encryption
  (`vibe/core/auth/crypto.py`, RSA-OAEP + AES-GCM).
* API keys are stored in `.env` inside `VIBE_HOME` (plain text — see
  business rule BR-002 in `docs/use_cases/UC-001-...`).

`[OPEN]`

* *Category*: quality-goals
* *Ask role*: Security
* *What is missing*: A formal threat model. In particular: handling of

  redaction config is visible).

== Q5 — What risks and technical debt exist?

=== Q5.1 — Concentration risk: god module

`[ANSWERED]` `vibe/core/agent_loop.py` is 1716 lines and contains
`class AgentLoop` with deferred-init machinery and thread-locked state
(`vibe/core/agent_loop.py:221-410`). Most changes to the engine pass
through this file.

=== Q5.2 — Pinned dependencies on fast-moving protocols

`[ANSWERED]` `agent-client-protocol==0.9.0` and `textual==8.2.4` are
pinned to exact versions; `mistralai==2.4.4` is pinned exactly. ACP is
a young protocol and Textual ships frequent breaking releases. Drift
between Mistral SDK and Mistral API will block upgrades.

`[ANSWERED]` Three explicit "legacy" markers:

* `vibe/cli/update_notifier/adapters/filesystem_update_cache_repository.py`
  — legacy migration logic for older update-cache schema.
* `vibe/core/llm/backend/anthropic.py` — referenced as `legacy` in
  comments; status unclear.

`[OPEN]`

* *Category*: future-direction
* *Ask role*: Architect / Developer
