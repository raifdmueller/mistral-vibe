== 1. Introduction and goals (Q3.1)

Mistral Vibe is Mistral AI's open-source command-line coding
assistant (`README.md:24`). It exposes a conversational interface to
a codebase through a Textual TUI, a non-interactive `-p` mode, and an
Agent Client Protocol (ACP) server. The codebase targets Python ظëح
3.12 (`pyproject.toml:6`) and is officially supported on Linux and
macOS (`README.md:25-26`).

`(open ظ¤ see Q1.5)` Quantitative product goals and KPIs are not
declared in this repository.

== 4. Solution strategy (Q3.4)

* *Hexagonal-style ports*: abstract interfaces use the `_port.py`
  suffix (`AGENTS.md:24`).
* *Pydantic-first data*: external data is validated with `BaseModel`
  / `pydantic-settings` (`AGENTS.md:51-58`,
  `vibe/core/config/_settings.py:498`).
* *Stream-by-default*: tool and model events are
  `AsyncGenerator[Event, None]` (`AGENTS.md:67`).
* *asyncio orchestration*: structured with `asyncio.create_task` and
  queues, not blanket `gather` (`AGENTS.md:64-66`).
* *Single Mistral provider default, OpenAI-style fallback*: every
  third-party provider is treated as an OpenAI-style endpoint
  (`vibe/core/config/_settings.py:172-218`).
* *Single source of truth for config*: `VibeConfig` overlays
  defaults ظْ `config.toml` ظْ `VIBE_*` environment variables
  (`vibe/core/config/_settings.py:648-651`).

`(open ظ¤ see Q3.4)` Recorded rationale for the foundational choices
(Textual, `uv`, ACP 0.9.0 specifically) is missing ظ¤ see ADR backlog
in `ADRs.adoc`.

=== 5.2 Level 2 ظ¤ engine internals (`vibe/core/`)

* `agent_loop.py` orchestrates: input ظْ tool registry ظْ LLM call ظْ
  permission check ظْ tool execution ظْ event stream ظْ log append.
* `tools/base.py` defines `BaseTool`, `BaseToolConfig`, `ToolPermission`,
  `ToolError`, `ToolPermissionError`.
* `tools/manager.py` aggregates builtin tools, custom tools, MCP
  tools, and connector tools into the active set.
* `tools/permissions.py` defines permission scopes
  (`COMMAND_PATTERN`, `OUTSIDE_DIRECTORY`, `FILE_PATTERN`,
  `URL_PATTERN`) and the approval data model.
* `agents/manager.py` resolves the active `AgentProfile` and applies
  its overrides on top of `VibeConfig`.
* `config/_settings.py` defines all configuration models and the
  `VibeConfig` `BaseSettings` aggregate.
* `session/session_logger.py` writes `meta.json` + `messages.jsonl`
  in a session directory under `session_logging.save_dir`.
* `hooks/` discovers and executes post-turn hooks.
* `llm/backend/` contains the active set of provider backends
  (`mistral`, `anthropic` ظ¤ the latter marked legacy).

== 6. Runtime view (Q3.6)

=== 6.1 Conversation turn

. Surface (CLI / ACP) receives input.
. `AgentLoop` composes the system prompt
  (`vibe/core/config/_settings.py:725-734`).
. `ToolManager` produces the available tool list.
. LLM backend is called with messages + tools (streamed).
. For every tool call in the response:
.. permission resolved via `ToolPermission` and active
   `AgentProfile`;
.. if `ASK`, the surface prompts the user (or, on ACP, asks the
   client);
.. on approval, the tool runs and streams events back;
.. result is appended to the conversation.
. Steps 4ظô5 loop until the assistant produces a final message with
  no tool calls.
. Post-turn hooks run (`vibe/core/hooks/`).
. Messages and metadata are appended to the session log.

Auto-compaction fires when input-token count crosses the active
model's `auto_compact_threshold`
(`vibe/core/config/_settings.py:374`, default 200 000).

=== 6.2 Startup

* `vibe/cli/entrypoint.py:166` parses arguments, resolves
  `--workdir`, applies `--trust`, then dispatches to `run_cli`.
* `check_and_resolve_trusted_folder`
  (`vibe/cli/entrypoint.py:124-153`) gates the trust dialog
  (UC-006).
* `init_harness_files_manager("user", "project")`
  (`vibe/cli/entrypoint.py:158`) registers project- and user-scope
  override paths.
* `AgentLoop` may defer MCP and connector discovery onto a daemon
  thread (`vibe/core/agent_loop.py:234,348-356`,
  `defer_heavy_init`).

=== 6.3 Teleport

`/teleport` packages the working tree, conversation, and todos and
ships them through the Nuage workflow client
(`vibe/core/nuage/client.py`,
`vibe/core/agent_loop.py:162` `TeleportError`).

=== 6.4 Voice and TTS

* Voice input streams 16 kHz PCM to the Transcribe model with a
  500 ms target latency
  (`vibe/core/config/_settings.py:380-388`).
* TTS streams the assistant's reply through the TTS model
  (`vibe/core/tts/`).

== 7. Deployment view (Q3.7)

* *Local install*: `uv tool install mistral-vibe`,
  `pip install mistral-vibe`, or the install script
  (`README.md:36-58`).
* *Process model*: single Python process for CLI; ACP server runs
  as a separate process spawned by the IDE
  (`vibe/acp/entrypoint.py`).
* *CI*: `action.yml` runs `vibe -p` in a GitHub Actions runner.
* *Packaging*: PyInstaller spec for redistributable binaries
  (`vibe-acp.spec`, `pyinstaller/`). `flake.nix` provides a Nix dev
  shell.

`(open ظ¤ see Q4.8)` Whether PyInstaller binaries are an officially
shipped artifact, and the practical Windows / ARM64 support contract,
are not declared.

== 11. Risks and technical debt (Q5)

=== 11.1 Known

* *Concentration risk*: `vibe/core/agent_loop.py` is 1716 lines and
  is the de facto god module (Q5.1).
* *Pinned dependencies on young protocols*:
  `agent-client-protocol==0.9.0`, `textual==8.2.4`,
  `mistralai==2.4.4` (Q5.2).
* *Three explicit "legacy" markers* (Q5.3):
  - `vibe/core/config/_settings.py:185` ظ¤ legacy Mistral provider
    without backend;
  - `vibe/cli/update_notifier/adapters/filesystem_update_cache_repository.py`
    ظ¤ legacy migration logic;
  - `vibe/core/llm/backend/anthropic.py` ظ¤ referenced as legacy.
* *Committed TODOs*:
  `vibe/core/config/_settings.py:530` (OTEL public-feature gate),
  `vibe/core/autocompletion/completers.py:330` (root-directory
  assumption) (Q5.4).
* *`keyring` dependency vs `.env` storage*: `keyring>=25.6.0` is
  declared but no `keyring.set_password` is reachable from the
  onboarding path; API keys are written to `~/.vibe/.env` in plain
  text (Q5.5).
* *Hidden roadmap flags*: `vibe_code_*`, `enable_otel`,
  `otel_endpoint`, `enable_experimental_hooks` are excluded from
  public serialisation (`vibe/core/config/_settings.py:524-535`)
  (Q5.7).
