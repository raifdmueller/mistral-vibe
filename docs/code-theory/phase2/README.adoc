== How `(team answer)` works

The workflow asks Phase 2 to mark team-provided information with
`(team answer)`. Until the team responds, this document uses
`(open — see Q-X.Y)` instead. When answers arrive, every
`(open — …)` marker should be replaced by `(team answer)` plus the
text the team provided.

== Baseline tests

The workflow recommends a baseline test suite tied to use case IDs.
The Gherkin scenarios in `acceptance.feature` are that backbone; the
mapping table (UC → existing pytest modules in `tests/`) is
`(open — see Q2.6)` until the team confirms which modules they
consider acceptance tests.
