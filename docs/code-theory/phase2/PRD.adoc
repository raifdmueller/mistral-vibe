== 1. Product summary

Mistral Vibe is an open-source command-line coding assistant powered
by Mistral's models. It provides a conversational interface to a
codebase: a developer types natural-language requests in a terminal
and the assistant uses a built-in tool set (file read/write, shell,
grep, web fetch, web search, todo, task) to explore and modify the
project (Q1.1).

== 4. Users and stakeholders

=== 4.1 Primary users (Q1.2)

* *Developers using a UNIX terminal* — Linux and macOS officially
  supported; Windows works but is unsupported (`README.md:25-26`).
  The product is classified `Intended Audience :: Developers`
  (`pyproject.toml:18`).

=== 4.2 Secondary users (Q1.2, Q1.3)

* *Automation / CI* — invoked through the programmatic mode
  (`vibe -p`, `vibe/cli/entrypoint.py:41-49`) or via the bundled
  GitHub Action (`action.yml`).
* *IDE users* — drive the assistant indirectly through an Agent
  Client Protocol client (Zed, JetBrains AI Assistant, Neovim
  avante.nvim; `docs/acp-setup.md`).

=== 4.3 Non-user stakeholders

`(open — see Q1.6)` The repository identifies "Mistral AI" as the
author (`pyproject.toml:8`) but not the internal stakeholders for
product direction, operations, Mistral API capacity, or the Vibe Code
teleport endpoint.

== 9. Glossary (partial — see Q3.12)

`Session` :: A single persisted conversation
(`vibe/core/types.py:146-160`).
`Agent profile` :: A bundle of behaviour overrides selecting tools
and safety class (`vibe/core/agents/models.py:49`).
`Skill` :: A reusable Markdown prompt invoked through the slash menu
(`vibe/core/skills/models.py:9`).
`Tool` :: A callable surface exposed to the model
(`vibe/core/tools/base.py`).
`Hook` :: A shell command run after every assistant turn
(`vibe/core/hooks/models.py:26`).
`Trusted folder` :: A directory whose agent-behavior files are
allowed to influence the system prompt
(`vibe/core/trusted_folders.py`).
`Teleport` :: Handoff of a local session to the Vibe Code workflow
service (`vibe/core/teleport/`, `/teleport` command in
`vibe/cli/commands.py`).
`Subagent` :: An agent profile invocable only via the `task` tool
(`vibe/core/agents/models.py:34`).

`(open — see Q3.12)` Whether terms like "Vibe Code", "Voxtral",
"Codestral", "leanstral", "Nuage", and "teleport" are public product
names, internal codenames, or both is not recorded.

== 10. Data and privacy

* Telemetry is opt-out (`enable_telemetry: bool = True`,
  `vibe/core/config/_settings.py:519`).
* Session logging is opt-out
  (`vibe/core/config/_settings.py:556`).
* Local data lives in the Vibe home directory (default `~/.vibe`,
  override via `VIBE_HOME`; `vibe/cli/entrypoint.py:23-30`).

`(open — see Q5.6)` Whether telemetry payloads have been audited
for PII / secret leakage is undocumented.

`(open — see Q1.5)` Retention duration of session logs and
telemetry on the remote side is not declared in this repository.
