== 3. CLI specification (Q2.3)

Defined by the argparse spec at `vibe/cli/entrypoint.py:19-122`.

=== 3.1 Synopsis

----
vibe [PROMPT] [options]
----

=== 3.2 Options

[cols="1,2,3", options="header"]
|===
| Flag | Value | Effect

| `PROMPT` | string (positional, optional)
| Initial user message for an interactive session.
| `-p, --prompt` | string (optional value)
| Programmatic mode: send the prompt, auto-approve all tools, write
  the final answer to stdout, exit.
| `--max-turns` | integer
| Cap on assistant turns (programmatic only).
| `--max-price` | decimal (USD)
| Cap on total session cost (programmatic only). Session aborts when
  exceeded.
| `--enabled-tools` | name / glob / `re:`-regex (repeatable)
| Restrict the active tool set. In programmatic mode this also
  disables every other tool.
| `--output` | `text` \| `json` \| `streaming`
| Output format for programmatic mode.
| `--agent` | profile name
| Override the active agent profile.
| `--setup` | flag
| Run onboarding (API key) and exit.
| `--workdir` | path
| `chdir` into this directory before doing anything else.
| `--trust` | flag
| Trust the working directory for this invocation only — not
  persisted to `trusted-folders.toml`.
| `--teleport` | flag (hidden — `argparse.SUPPRESS`)
| Internal feature flag.
| `-c, --continue` | flag
| Resume the most recent saved session for the current working
  directory.
| `--resume [SESSION_ID]` | flag with optional value
| Open the resume picker, or resume the named session directly.
| `-v, --version` | flag
| Print the package version.
|===

=== 3.3 Environment variables

`VIBE_HOME` :: Override the Vibe home directory (default `~/.vibe`).
`LOG_LEVEL` :: `DEBUG` \| `INFO` \| `WARNING` (default) \| `ERROR`
\| `CRITICAL`. Logs land in `$VIBE_HOME/logs/vibe.log`.
`LOG_MAX_BYTES` :: Max size of `vibe.log` before rotation (default
`10485760`).
`VIBE_*` :: Override any `VibeConfig` field (for example
`VIBE_ACTIVE_MODEL=local`;
`vibe/core/config/_settings.py:648-651`).

=== 3.4 Exit status

`0` :: Successful completion.
non-zero :: Budget exceeded (`--max-turns`, `--max-price`),
authentication error, unrecoverable model error, invalid arguments,
or non-existent `--workdir`. (See UC-013.)

== 4. ACP specification (Q2.4)

Defined by the handler methods in `vibe/acp/acp_agent_loop.py` and
`vibe/acp/session.py`. Session lifecycle methods: `initialize`,
`new_session`, `load_session`, `fork_session`, `resume_session`,
`close_session`, `list_sessions`. Execution methods: `prompt`,
`cancel`. Configuration methods: `set_session_mode`,
`set_session_model`, `set_config_option`. Tool surface is the
ACP-side wrapper set under `vibe/acp/tools/builtins/`.

`(open — see Q3.4)` The protocol version contract — what range of
ACP versions is supported beyond the pinned `0.9.0`
(`pyproject.toml`) — is not declared.

== 6. Acceptance criteria

Gherkin features live in `acceptance.feature` next to this document.
Each scenario references a use case ID for traceability (see Phase 2
"baseline tests" guidance in the source workflow).

`(open — see Q2.6)` Whether any pytest modules under `tests/` are
formally considered acceptance tests (rather than unit / integration
tests) is not declared.

== 7. Cross-cutting business rules (Q2.7)

The 58 business rules BR-001..BR-058 are stated in
`../../use_cases/UC-*.md`. They cover permissions, tool truncation,
auto-compaction, session persistence, trust scoping, hook execution,
skill validation, model selection, programmatic-mode defaults, and
ACP-tool parity with interactive mode.

`(open — see Q2.7)` Whether telemetry redaction, model cost caps,
supported model matrix, retention duration of session logs, and
"what counts as a destructive tool" are formal rules with versioned
approvals or implicit conventions cannot be derived from code.
