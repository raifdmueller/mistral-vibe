== Introduction and Goals

Mistral Vibe is Mistral's open-source CLI coding assistant — a conversational
agent that runs in a developer's terminal, reads and writes files, executes
shell commands, searches code, and calls external tools, all driven by Mistral
models (`README.md:22`, `pyproject.toml:4`).

Top three quality goals (derived; ranking is `[OPEN — see Q4.9]`):

* *Security* — an LLM may propose destructive actions, so tool execution is
gated and the shell is restricted (`vibe/core/tools/permissions.py`,
`vibe/core/tools/builtins/bash.py:120-147`).
* *Reliability* — transient LLM/network failure must not crash a session
(`vibe/core/utils/retry.py`).
* *Extensibility* — new tools arrive via MCP, connectors, and skills without
code changes (`vibe/core/tools/mcp/`, `vibe/core/skills/`).

Stakeholders: the Developer (user), Mistral (vendor), open-source contributors
(`CONTRIBUTING.md`).

== Solution Strategy

Vibe is a layered Python package. `vibe/core` holds the domain and the agent
loop, free of any UI or provider detail. `vibe/cli` is the Textual TUI.
`vibe/acp` is the Agent Client Protocol server for editors. `vibe/setup` handles
onboarding and auth.

Integration seams use ports-and-adapters: a `ports/` package defines an abstract
contract and an `adapters/` package supplies the concrete implementation
(`vibe/cli/update_notifier/ports/` + `adapters/`, likewise for `plan_offer`,
`narrator_manager`, `voice_manager`, `notifications`). The code is async
end-to-end (`anyio`, `asyncio`). Five LLM backends sit behind one factory so the
agent loop never knows which provider it talks to
(`vibe/core/llm/backend/factory.py`).

Vibe is distributed three ways: a PyPI wheel installed by `uv tool install` /

== Crosscutting Concepts

=== 8.1 Threat Model (STRIDE)

The trust boundary: an LLM (not fully trusted) drives tool calls that touch the
user's shell and filesystem; project folders may be hostile.

[cols="1,1,3,2",options="header"]
|===
| T-ID | STRIDE | Threat | Code-visible exposure

| T-001 | Tampering | LLM proposes a destructive shell command | `bash` tool runs
arbitrary commands (`bash.py`)
| T-002 | Elevation of Privilege | User runs under `auto-approve` /
`bypass_tool_permissions`, removing the approval gate | `_settings.py:510`
| T-003 | Tampering | A hostile project folder's `AGENTS.md` / `config.toml`
injects instructions or settings | `trusted_folders.py`,
`vibe/core/paths/_local_config_walk.py`
| T-004 | Information Disclosure | API keys or tokens leak via config files or
logs | `auth/github.py`, `vibe/core/logger.py`
| T-005 | Tampering | An MCP child process inherits env and runs arbitrary code
| `_settings.py:315-324` (`MCPStdio` env/cwd passthrough)
| T-006 | Spoofing / Tampering | Tampered remote workflow payload during teleport
| `vibe/core/nuage/`, `auth/crypto.py`
| T-007 | Denial of Service | A hung LLM call blocks the session up to 720 s |
`_settings.py:520` `api_timeout`
| T-008 | Information Disclosure | Telemetry on by default exfiltrates usage data
| `_settings.py:511` `enable_telemetry`
|===

=== 8.2 Security

Each mitigation cites the threats it closes.

* *Tool-approval gate* — every tool call requires explicit approval unless the
profile auto-approves (`tools/permissions.py`). Closes T-001 (partially), gates
T-002.
* *Shell denylist* — interactive shells, editors, and standalone interpreters
are rejected; path commands are confined to the working directory
(`bash.py:120-147`, `tools/utils.is_path_within_workdir`). Closes T-001.
* *Trusted-folder gate* — project-level config loads only from a folder the user
has trusted (`trusted_folders.py`, UC-021). Closes T-003.
* *Keyring credential storage* — credentials live in the OS keyring, not
plaintext config (`auth/github.py` via `keyring`). Closes T-004.
* *Payload encryption* — teleport payloads use AES-256-GCM with an RSA-OAEP-
SHA256-wrapped key, RSA key >= 2048 bits, ciphertext capped at 2 MiB,
encrypted-key capped at 1 KiB (`auth/crypto.py:13-17`). Closes T-006.
* *Request timeout* — `api_timeout` bounds a hung LLM call (`_settings.py:520`).
Mitigates T-007.
* *Telemetry opt-out* — `enable_telemetry` is a config flag the user can disable
(`_settings.py:511`). Mitigates T-008.

T-005 (MCP child-process trust) has no code-level mitigation beyond the user
choosing which servers to configure — `[OPEN — see Q5.2 / Operations]`.

=== 8.3 Test

The testing pyramid: 228 test modules under `tests/`.

* *Unit* — config, message merging, history, system prompt, tracing
(`tests/test_*.py`).
* *Tool* — per-tool behaviour and permissions (`tests/tools/`).
* *Component / manager* — narrator, voice, transcribe (`tests/narrator_manager/`,
`tests/voice_manager/`, `tests/transcribe/`).
* *Snapshot* — pinned TUI rendering (`tests/snapshots/`, 25+ modules).
* *e2e* — full CLI runs in a forked process (see `pyproject.toml` filterwarnings).

CI runs `uv run pytest --ignore tests/snapshots` then the snapshot suite
separately (`.github/workflows/ci.yml:87,116`). Per-test timeout 10 s
(`pyproject.toml` tool.pytest). Tests trace to use cases and business rules via
the Gherkin scenarios in `docs/specification.adoc`.

=== 8.4 Observability

* *Tracing* — OpenTelemetry spans via `vibe/core/tracing.py` and the
`opentelemetry-*` dependency set; an OTLP exporter is configurable
(`OtelSpanExporterConfig`, `_settings.py:413`).
* *Logging* — structured logging through `vibe/core/logger.py`; a log reader
(`vibe/core/log_reader.py`) and an in-app debug console
(`vibe/cli/textual_ui/widgets/debug_console.py`).
* *Metrics / audit* — per-session `AGENT_STATS` (steps, tokens, cost, tool
outcomes) and telemetry events (`vibe/core/telemetry/`); session logs on disk
form an audit trail (`vibe/core/session/session_logger.py`).

=== 8.5 Error Handling

* *Retry* — `async_retry` / `async_generator_retry`: 3 tries, 0.5 s initial
delay, ×2 backoff, retryable HTTP statuses 408/409/425/429/500/502/503/504/529
(`utils/retry.py:11,17`).
* *Error classification* — `_should_raise_rate_limit_error` and
`_is_non_retryable_error` (walking `__cause__`) decide whether to retry or
surface (`agent_loop.py:166-187`).
* *Timeout / fallback* — bash timeout yields a `ToolError` rather than a crash
(`bash.py:460`); MCP startup/tool timeouts disable the affected server.
* *Recovery* — sessions persist to disk and can be resumed; session migration
upgrades old formats (`session/resume_sessions.py`, `session/session_migration.py`).

No explicit circuit breaker was found — retry plus error classification is the
resilience pattern in use.

=== Quality Tree

* Security — tool gating, shell denylist, trust, crypto.
* Reliability — retry, error classification, session recovery.
* Performance Efficiency — timeouts and context budget.
* Extensibility / Maintainability — ports-and-adapters, lint gates.
* Compatibility — five backends, three MCP transports, ACP.
* Portability — three distribution forms.

== Glossary

[cols="1,3",options="header"]
|===
| Term | Definition
| Agent Loop | The turn cycle: prompt -> completion -> tool calls -> results ->
repeat (`agent_loop.py`).
| Agent Profile | A named bundle of tool permissions defining how autonomous the
agent is (`default`, `plan`, `accept-edits`, `auto-approve`).
| Tool | A capability the agent invokes — builtin, MCP-provided, or
connector-provided.
| MCP Server | An external Model Context Protocol server whose tools are exposed
to the agent.
| Connector | A managed integration exposing tools, identified by alias.
| Skill | A packaged set of instructions and optional pre-approved tools, also
usable as a slash command.
| Session | One conversation, optionally persisted to disk for resume.
| Compaction | Summarising the conversation when context exceeds the model
threshold.
| Trusted Folder | A working directory the user has allowed project-level config
to load from.
| ACP | Agent Client Protocol — the editor/IDE integration interface.
| Vibe Code / Nuage | The remote workflow backend a session can teleport to.
| Thinking level | The reasoning-effort setting (off / low / medium / high / max).
| Hook | A user shell command run automatically at a lifecycle event
(`post_agent_turn`).
|===
