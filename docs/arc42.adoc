== 2. Architecture Constraints

* Python ‚â• 3.12 (`pyproject.toml:21`).
* Dependency and task management exclusively via `uv` (`AGENTS.md` "Commands").
* UNIX officially supported; Windows best-effort (`README.md:24-25`).
* Apache-2.0 license (`LICENSE`).
* Mandatory toolchain: `ruff`, `pyright` (strict), `vulture`, `pre-commit`
  (`pyproject.toml:103-156`, `.pre-commit-config.yaml`).
* Coding conventions: modern type hints, `match`/`case`, early returns,
  `pathlib`, `_port.py` interfaces, `models.py` for Pydantic models
  (`AGENTS.md` "Python style", "Project layout").

== 4. Solution Strategy

* *Layered modules:* an engine (`vibe/core`) separated from delivery channels
  (`vibe/cli`, `vibe/acp`, `vibe/setup`) (`AGENTS.md:5`).
* *Hexagonal ports/adapters:* abstract interfaces use the `_port.py` suffix
  with concrete `adapters/` (`AGENTS.md` "Project layout").
* *Async core:* `asyncio` / `anyio` throughout the agent loop and I/O.
* *Textual TUI* for the interactive experience (`pyproject.toml:48`).
* *Layered, merge-aware configuration* (`vibe/core/config/`).

The rationale behind these strategic choices is recorded as open in
`docs/architecture-decisions.adoc`.

== 6. Runtime View

A turn proceeds as: the developer submits input ‚Üí project context is gathered
‚Üí the request is sent through a selected LLM backend ‚Üí the agent proposes tool
calls ‚Üí each call passes the permission/approval check ‚Üí results are fed back ‚Üí
the loop repeats until the agent yields a final answer.

* The turn loop runs each step through a `MiddlewarePipeline`
  (`vibe/core/agent_loop.py:281-282`, `vibe/core/middleware.py:218`), which can
  short-circuit on turn limit, price limit, auto-compaction, or read-only mode
  (`vibe/core/middleware.py:49-99,174`).
* Heavy initialization is deferred to a background thread to speed startup
  (`vibe/core/agent_loop.py:241-246`, `:349-354`).
* Scheduled loops are polled and fired by `ScheduledLoopRunner`
  (`vibe/cli/textual_ui/scheduled_loop_runner.py`).

== 7. Deployment View

* *Installation:* `uv tool install mistral-vibe`, `pip install mistral-vibe`,
  or the one-line `scripts/install.sh` (`README.md:27-55`).
* *Standalone binaries:* built with PyInstaller (`pyinstaller/`,
  `vibe-acp.spec`, `distribution/`).
* *CI usage:* a GitHub composite Action installs Python + `uv`, syncs the
  project, and runs `vibe -p` (`action.yml`).
* *Entrypoints:* two console scripts, `vibe` and `vibe-acp`
  (`pyproject.toml:73-75`).
* *Home directory:* state lives in `~/.vibe/` (config, `.env`, agents, prompts,
  tools, logs), relocatable via `VIBE_HOME` (`README.md:632-647`).

== 10. Quality Requirements

Quality mechanisms present in code, mapped to ISO/IEC 25010 characteristics:

[cols="1,3"]
|===
| Characteristic | Mechanism (evidence)

| Functional Suitability | Broad `tests/` suite (unit, snapshot, e2e).
| Performance Efficiency | `textual-speedups` (`pyproject.toml:49`); deferred init (`agent_loop.py:241`); async I/O.
| Compatibility | MCP, ACP, and Agent Skills standards (`README.md:344`).
| Usability | Textual TUI, autocompletion (`vibe/cli/autocompletion/`), onboarding wizard.
| Reliability | Retry helper (`vibe/core/utils/retry.py`); session logging; typed errors.
| Security | Trust folders, tool approval, `keyring`/`.env` secret storage.
| Maintainability | `ruff`, `pyright` strict, `vulture`, `pre-commit`; `AGENTS.md` conventions.
| Portability | UNIX-first + Windows; `VIBE_HOME`; `flake.nix`.
|===

== 12. Glossary

[cols="1,3"]
|===
| Term | Meaning

| Agent | A behavior profile that bounds what the assistant may do without approval (`vibe/core/agents/models.py`).
| Subagent | An agent delegated a scoped task via the `task` tool (`vibe/core/tools/builtins/task.py`).
| Tool | A discrete capability the agent can invoke (`vibe/core/tools/`).
| Skill | A packaged extension adding instructions / slash commands (`vibe/core/skills/`).
| MCP | Model Context Protocol ‚Äî external tool servers (`vibe/core/tools/mcp/`).
| ACP | Agent Client Protocol ‚Äî the IDE integration channel (`vibe/acp/`).
| Teleport / Vibe Code | Handing a session to a Mistral cloud workflow (`vibe/core/teleport/`).
| Session | A recorded, resumable conversation (`vibe/core/session/`).
| Harness files | Resolved config/instruction files for a run (`vibe/core/config/harness_files/`).
| Connector | A managed external tool provider (`vibe/core/tools/connectors/`).
|===
