=== SUC-1 — `vibe` CLI

Entry point `vibe.cli.entrypoint:main` (`pyproject.toml:87`). Argument surface
(`vibe/cli/entrypoint.py:19-126`):

[cols="1,3",options="header"]
|===
| Flag | Effect
| `PROMPT` (positional) | Initial prompt for an interactive session.
| `-p, --prompt [TEXT]` | Programmatic mode; auto-approves tools, prints, exits.
| `--max-turns N` | Turn cap (programmatic only).
| `--max-price DOLLARS` | Cost cap (programmatic only).
| `--enabled-tools TOOL` | Tool allowlist; exact / glob / `re:` regex.
| `--output {text,json,streaming}` | Programmatic output format.
| `--agent NAME` | Builtin or custom agent profile.
| `--setup` | Configure API key and exit.
| `--workdir DIR` | Change directory before running.
| `--trust` | Trust the cwd for this invocation only.
| `-c, --continue` | Resume the most recent session.
| `--resume [SESSION_ID]` | Resume a session; picker if no ID.
| `--teleport` | Hidden feature flag (`argparse.SUPPRESS`).
|===

Validation seen in code: `--workdir` must be an existing directory, else exit 1
(`vibe/cli/entrypoint.py:160-167`); a deleted cwd exits 1
(`vibe/cli/entrypoint.py:169-178`).

=== SUC-2 — `vibe-acp` JSON-RPC server

Entry point `vibe.acp.entrypoint:main` (`pyproject.toml:88`). Implements the
Agent Client Protocol (`agent-client-protocol==0.9.0`) so an IDE can drive the
agent. Session and tool handling in `vibe/acp/session.py`,
`vibe/acp/acp_agent_loop.py`. Per-method request/response contracts are
`[OPEN — see Q2.3]`.

=== SUC-3 — Configuration file formats

`config.toml` (project `./.vibe/` then `~/.vibe/`), `agents/*.toml`, and the
`mcp_servers` table. Schema and validation in `vibe/core/config/schema.py`;
layering and the trust gate in `vibe/core/config/layer.py`. Untrusted layers
raise `UntrustedLayerError` (`vibe/core/config/layer.py:25-29`).

=== SUC-4 — Session-log files

Conversations are persisted to disk and resumed
(`vibe/core/session/session_logger.py`, `session_loader.py`,
`resume_sessions.py`). A migration path exists (`session_migration.py`). The
on-disk format contract is `[OPEN — see Q2.3]`.

== Individual Requirements (EARS)

Recovered from code, expressed in EARS:

* *REQ-1* — When the developer runs `vibe` in a directory containing `.vibe`
config and trust status is unresolved, the system shall prompt for trust before
loading project-local configuration (`vibe/cli/entrypoint.py:129-155`).
* *REQ-2* — While in programmatic mode, if the accumulated cost exceeds
`--max-price`, the system shall interrupt the session
(`vibe/core/middleware.py`, `PriceLimitMiddleware`).
* *REQ-3* — If a tool action does not match an allow rule and is not bypassed,
the system shall request developer approval before running it (UC-008, BR-013).
* *REQ-4* — When a config layer is untrusted, the system shall refuse to load
its data and raise `UntrustedLayerError` (`vibe/core/config/layer.py:25-29`).
