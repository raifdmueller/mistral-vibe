== Problem

Developers work with their codebase from the terminal but switch to other
tools to get AI assistance. Mistral Vibe removes that switch: it is a
command-line coding assistant that gives a conversational, natural-language
interface to a codebase, so a developer can explore and modify a project
through a set of tools without leaving the shell (`README.md:21-23`).

== Personas

The code exposes three distinct user roles through its entry points.

Interactive Developer::
Runs `vibe` in a terminal and works with the agent through a Textual TUI,
optionally passing a starting prompt and working directory
(`vibe/cli/entrypoint.py:19-119`).

Automation / Script::
Runs `vibe -p` (programmatic mode): one prompt, all tools auto-approved,
result on stdout, then exit. Used in CI through the bundled GitHub Action
(`vibe/cli/entrypoint.py:42-49`, `action.yml`).

IDE Integrator::
Drives Mistral Vibe from an editor over the Agent Client Protocol by running
`vibe-acp` (`vibe/acp/entrypoint.py:30-37`). A Zed extension ships in
`distribution/zed/`.

== Scope

=== In scope (observed in code)

The product delivers fifteen user-facing capabilities, fully specified in
`docs/use_cases/` (UC-001 … UC-015) and summarized in `docs/use_cases.puml`:

* Interactive coding sessions with an LLM-backed agent and a tool set
  (`vibe/core/agent_loop.py:221`).
* Authentication via API key or browser sign-in
  (`vibe/setup/auth/browser_sign_in.py`).
* Agent modes from read-only planning to fully auto-approved execution
  (`vibe/core/agents/models.py:108-188`).
* Session persistence, resume, rename, compaction, and rewind
  (`vibe/core/session/`, `vibe/core/rewind/`).
* Extension via MCP servers, connectors, skills, and hooks
  (`vibe/core/tools/mcp/`, `vibe/core/skills/`, `vibe/core/hooks/`).
* Non-interactive programmatic mode and ACP-based IDE integration
  (`vibe/core/programmatic.py`, `vibe/acp/`).
* Voice input and spoken narration; teleport to Vibe Code Cloud
  (`vibe/core/transcribe/`, `vibe/core/tts/`, `vibe/core/teleport/`).

== Constraints

* Supported platform is UNIX; Windows is best-effort, not a target
  (`README.md:25-26`).
* Python ≥ 3.12, `uv`-managed (`pyproject.toml`, `.python-version`).
* Licensed Apache-2.0 (`LICENSE`).
